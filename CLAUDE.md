## Coordinate system (robot-car)

**Convention:** −X = front/bumper · +X = rear · −Y = left · +Y = right · +Z = up

This differs from standard ROS (+X forward, +Y left). Our robot faces −X, so the
right-hand rule gives left = Z × (−X) = **−Y**. Don't assume +Y = left.

`_applyUp('+Z')` applies `world.rotation.set(-PI/2, 0, 0)`, mapping:

| URDF axis | Three.js axis |
|-----------|--------------|
| X         | X            |
| Y         | −Z           |
| Z         | +Y (up)      |

Default camera is placed along direction `(-1, 0.7, 1)` from the robot center,
putting it at roughly URDF (−5.5, −5.5, +3.5) — the front-left corner viewed from above.

**Before editing any joint position, verify screen placement with this snippet:**

```python
import numpy as np

def screen_x(urdf_x, urdf_y, urdf_z):
    """Positive = screen right, negative = screen left."""
    tx, ty, tz = urdf_x, urdf_z, -urdf_y   # R_x(-PI/2)
    right = np.array([1, 0, 1]) / np.sqrt(2)
    return float(np.dot([tx, ty, tz], right))

# Example
print(screen_x(-0.010, -0.090, -0.002))  # wheel_left → negative = screen left ✓
print(screen_x(-0.010,  0.090, -0.002))  # wheel_right → positive = screen right ✓
```

## Sidebar UI conventions

The sidebar (`example/index.html`) is a **hybrid panel** — it overlays the 3D viewport and
is used on both desktop (mouse) and mobile (touch). Style it accordingly:

- The reference genre is desktop developer-tool (VS Code, Chrome DevTools, Three.js editor):
  compact, information-dense, small fonts, subdued chrome.
- Controls should be comfortably tappable on touch without looking like a mobile app.
  A good mental model is an iPad productivity app, not an iPhone app.
- Apple HIG 44pt touch targets are too large for this context and make it look bloated.
  The full 44pt rule is for primary mobile UI; secondary tool panels use smaller targets.

CSS variables for all colors and theme values live in `:root` at the top of the `<style>` block.
The sidebar has `overflow: hidden`; the scrollable region is `.sidebar-scroll` (flex: 1, overflow-y: auto),
which contains the joints list and inspector. Credits are a pinned footer via `flex-shrink: 0`.
