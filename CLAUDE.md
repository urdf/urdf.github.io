## Coordinate system (robot-car)

**Convention:** −X = front/bumper · +X = rear · −Y = left · +Y = right · +Z = up

Differs from ROS (+X forward, +Y left). Don't assume +Y = left.

`_applyUp('+Z')` applies `world.rotation.set(-PI/2, 0, 0)`, mapping:

| URDF axis | Three.js axis |
|-----------|--------------|
| X         | X            |
| Y         | −Z           |
| Z         | +Y (up)      |

Default camera direction `(-1, 0.7, 1)` from robot center (front-left, above).

**Before editing any joint position, verify screen placement:**

```python
import numpy as np

def screen_x(urdf_x, urdf_y, urdf_z):
    """Positive = screen right, negative = screen left."""
    tx, ty, tz = urdf_x, urdf_z, -urdf_y   # R_x(-PI/2)
    right = np.array([1, 0, 1]) / np.sqrt(2)
    return float(np.dot([tx, ty, tz], right))

# Example
print(screen_x(-0.010, -0.090, -0.002))  # wheel_left → negative = screen left ✓
print(screen_x(-0.010,  0.090, -0.002))  # wheel_right → positive = screen right ✓
```

## chassis.stl

Generated by `example/public/robots/robot-car/gen_chassis.cjs` (also `src/generators/chassis.ts` for Build tab).
Bumper arc: circle center (25, 0) mm, R = 85 mm. X-flipped so bumper faces -X.

### Earcut hole constraint

Every hole in `s.holes` must be entirely inside the outer boundary. Two zones:

| Zone | y range | x boundary |
|------|---------|-----------|
| Body | y < stepY (0.080) | ±bodyHW (0.062) |
| Rear | y > stepY (0.080) | ±rearHW (0.095) |

Hole outside its zone's boundary causes earcut to create a phantom triangle. Symptom: light-gray diagonal wedge covering the chassis. Fix: move hole into a wider zone or reduce its x extent.

## Resizable panel

W3C APG Window Splitter. `role="separator"` + `tabindex="0"`.
Min 260 px, max min(700 px, 60 vw), default 380 px. `localStorage` key `urdf-panel-w`.
**Do not** persist open/closed state. Hidden on mobile (`max-width: 768px`).

## Sidebar UI conventions

Desktop developer-tool genre (VS Code, Chrome DevTools).
Controls must be tappable on touch. Apple HIG 44pt targets are too large for this panel.
Typography: `0.6875rem` body labels, `10px` badges, `0.625rem` uppercase pill buttons.

CSS variables live in `:root`. Scrollable region is `.sidebar-scroll` (flex: 1, overflow-y: auto).

**Build tab structure:**
- Parametric groups use `<details class="build-section">` (zero-JS accordions).
- Action buttons in `.build-footer` (sticky, outside `.sidebar-scroll`).
- `#build-new-robot` hidden via `#build-active-header:not([hidden]) ~ #build-new-robot { display: none; }`
- Component cards: Position + Size always visible; advanced fields (Rotation, Joint, Axis, Limits, Preview)
  in `<details class="build-comp-advanced">` (collapsed for fixed joints; auto-opens on non-fixed).

**Chat pane:** Height 160 px (`--chat-h`, `localStorage` key `urdf-chat-h`). Appears in Editor and Build tabs only
(hidden via `body:not(.editor-open):not(.build-open)` CSS rule). `#library-panel` is ordered before the chat
elements in the DOM so all tab bodies are siblings above the chat pane — this keeps the chat anchored at the
bottom regardless of which tab is active. Placeholder text updates on tab switch: Editor → "Ask AI to edit
this URDF…", Build → "Ask AI to add or modify components…".

## Gesture mode (`example/gesture.ts`)

MediaPipe `@mediapipe/tasks-vision` (CDN). Lazy-loaded on first button click.

### Gesture dispatch — one gesture, one job

| Gesture | MediaPipe label | Action |
|---------|----------------|--------|
| Closed Fist | `Closed_Fist` | Orbit camera (wrist lm[0] delta) |
| Pointing Up | `Pointing_Up` **or landmark check** | Hover highlight + dwell select (fingertip lm[8]). Dual detection: ML label for upward, landmark geometry for other angles. |
| Open Palm | `Open_Palm` | Reset all joints (hold 1 s) |
| Thumbs Up | `Thumb_Up` | Confirm / Continue — resolves guide-mode pause. Fires `onThumbsUp` callback once per occurrence (resets on any other gesture). |
| Any + joint selected | any | Wrist roll drives joint angle |
| Two hands | any | Pinch/spread zoom |

### MediaPipe configuration

```typescript
minHandDetectionConfidence: 0.7   // 0.5 default causes false detections in cluttered scenes
minHandPresenceConfidence:  0.5
minTrackingConfidence:      0.5   // keep lower so fast moves don't drop tracking
// NO delegate: 'GPU' — gesture subgraph has CPU-only ops (mediapipe issue #4712)
```

### Recognition loop pattern

```typescript
// Guard: camera ~30fps, rAF ~60fps — skip duplicate frames
if (video.currentTime !== this._lastVideoTime) {
    this._lastVideoTime = video.currentTime;
    const result = recognizer.recognizeForVideo(video, performance.now());
    // performance.now() is correct for live camera (not video.currentTime * 1000)
}
```

### Signal smoothing: One Euro Filter

Raw landmarks jitter 5-10mm. One Euro Filter (Casiez et al., CHI 2012) on wrist position
adapts cutoff to velocity: `f_min` (1.0 Hz) smoothing when still, `beta` (0.007) responsiveness
to speed. Chosen over EMA because it avoids the latency/smoothing tradeoff.

### Key landmark indices

| Index | Landmark |
|-------|---------|
| 0 | Wrist |
| 5 | Index MCP |
| 8 | Index fingertip |
| 12 | Middle fingertip |

## Gesture design constraints

Max 5 general-purpose gestures (IJHCI 2024). `Thumb_Up` is a 6th, contextual-only gesture (guide mode pause) — it does not participate in the main dispatch except when the pause promise is pending. Do not add more general-purpose gestures without removing one.
Prefer iconic gestures (fist=grab, palm=release) over cultural emblems.

Dwell selection: 800ms (700-900ms range). 30px movement resets the timer.

Wrist roll comfortable range: +/-30-40 degrees from neutral. Delta-based mapping.

**Known fatigue issue:** Open_Palm hold for 1s is a sustained static pose. Consider replacing
with a dynamic gesture (e.g., fist-to-open snap).
